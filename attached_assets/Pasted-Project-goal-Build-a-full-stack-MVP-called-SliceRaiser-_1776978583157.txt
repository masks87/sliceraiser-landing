Project goal: Build a full-stack MVP called SliceRaiser — a global real estate and alternative investment platform where users browse investment opportunities and track a personal portfolio. I already have my own styling/design system in this project, so do not change any existing styles, Tailwind config, theme tokens, fonts, colors, or shadcn component variants. Only add the structure, logic, data, and wiring described below. Use my existing components and styles wherever a UI element is needed.

Architecture (pnpm monorepo)
Set this up as a pnpm workspace monorepo with the following layout:

artifacts/sliceraiser — React 19 + Vite frontend (the investor portal)
artifacts/api-server — Express 5 REST API
lib/api-spec — OpenAPI 3.1 YAML spec (single source of truth for the API)
lib/api-client-react — Auto-generated React Query hooks + TS types (generated by Orval from the OpenAPI spec)
lib/api-zod — Auto-generated Zod schemas (also from the OpenAPI spec)
lib/db — Drizzle ORM schema, migrations, and Postgres client
scripts/ — Node scripts for seeding the database
Use TypeScript 5.9 everywhere.

Tech Stack
Frontend: React 19, Vite, Tailwind CSS 4, shadcn/ui, Lucide React (icons), Recharts (charts), Wouter (router), TanStack Query (via Orval-generated hooks)
Auth: Clerk (@clerk/react on the frontend, @clerk/express on the backend)
Backend: Express 5, Pino (logger)
Database: PostgreSQL with Drizzle ORM
API contract: OpenAPI 3.1 + Orval (codegen) + Zod (runtime validation)
Database Schema (Drizzle, in lib/db)
investment_opportunities table:
id (primary key)
title
description
location (country/city)
assetType (e.g., Residential, Commercial, Farmland, Private Equity, etc.)
incomeType (e.g., Rental, Capital Appreciation, Dividend, Mixed)
riskLevel (Low | Medium | High)
expectedReturn (numeric, annualized %)
minimumInvestment (numeric)
targetAmount (numeric)
fundedAmount (numeric) — used to compute funding progress %
imageUrl
createdAt, updatedAt timestamps
user_portfolios table:
id (primary key)
clerkUserId (unique, indexed) — owner of the portfolio
snapshot (JSON column) — contains:
holdings: array of { opportunityId, units, investedAmount, currentValue }
allocation: array of { category, percentage } (for the pie chart)
returnsHistory: array of { date, value } (for the performance line chart)
totals: { totalInvested, totalCurrentValue, totalReturn, totalReturnPct }
createdAt, updatedAt
Provide Drizzle migrations and a db client exported from lib/db.

OpenAPI Spec (lib/api-spec/openapi.yaml)
Define these endpoints:

GET /api/healthz → { status: "ok" }
GET /api/opportunities → array of InvestmentOpportunity
GET /api/opportunities/{id} → single InvestmentOpportunity (404 if missing)
GET /api/portfolio → authenticated user's PortfolioSnapshot (401 if not signed in; auto-create a default seeded portfolio for the user on first call)
Define schemas for InvestmentOpportunity, PortfolioSnapshot, Holding, AllocationSlice, ReturnPoint, and Totals matching the DB schema above.

Add a codegen script (pnpm --filter @workspace/api-spec run codegen) that runs Orval to regenerate lib/api-client-react and lib/api-zod from the spec.

Backend (artifacts/api-server)
Express 5 server bound to process.env.PORT
Pino logger middleware
CORS configured to allow the frontend artifact's origin
@clerk/express middleware to validate the session token on protected routes
Implement all four endpoints from the spec
For GET /api/portfolio: read the Clerk user ID from the request, look up the row in user_portfolios, and if none exists, create a default seeded snapshot (sample holdings/allocation/returns) and return it
All DB access goes through lib/db
